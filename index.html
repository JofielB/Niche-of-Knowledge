How to Talk to Anyone : This reading is like a review of the book; How to Talk to Anyone by Leil Lowndes and in this reading you can find the things that the author of those notes thought that were more important from the book. The things that I found more interesting about all the advices that this book has to offer are the next ones:

  • It is important to know even how to smile. It is not recommended to always smile immediately when you know a new person, you have to wait for the perfect moment to give it a more genuine smile.

  • Eye contact is very important to always have eye contact and if you have a lot of problems with that you can watch the nose or the eyebrows of the person you are speaking to.

  • You need to be as relaxed as possible, don’t make unnecessary movements because it can lead to bad misunderstandings.

  • The conversation needs to flow first into your heart then you speak and if you don’t know how to answer you can ask about the last thing the other said to continue the conversation.

  • Wearing something unique can be a good icebreaker.

  • Match the mood of the ones you are speaking to, for example if they are too calm try to speak in a calmer way or if they are too excited try to transmit that same emotion when you speak.

  • No matter what you are talking about if you talk about it try to talk with excitement.

  • The more you know about the better the conversations will be. It is important to know about many topics and not only about the thing you like.

  • Thank you for…

  • More tips: https://sive.rs/book/HowToTalkToAnyone

Inheritance: Inheritance in OOP (Object-Oriented Programming) is the ability to inherit the method and attributes to another class for example if you have the class Persona and this class has the attribute of height all the classes that inherit from Persona will have the height attribute.

Composition: From my point of view the composition is kinda similar to the Inheritance because you use other class attributes and methods to fulfill a purpose but with the difference that the classes that you use are not attached to the classes that use them. For example, if you have the method or process of walk in the Persona class and you want to use that method in a class called Alumno you only instantiate the class Persona on the Alumno class and use the methods or attributes that you need in this way the Alumno class is not directed attached to the class Persona.

The 4 Pillars of OOP: Today I learned that there are 4 principles in the OOP paradigm. I have already known them but I didn’t know that they were called that way. These 4 pillars are like recommendations of how to take advantage of the OOP paradigm:

  • Abstraction: This consists in obtaining the more particular or general attributes from an object and creating a class in base of them. Is very important to just abstract the more essential attributes of a class to make it possible to use in many situations.

  • Encapsulation: The encapsulation make reference to the liberty that other classes have to access to the attributes and methods of the other classes, for example if you are using a class called people you will be able to access to its public methods or attributes, for example let's said that the class has a attribute name and is public you will be able to use it but if the class has an attribute called id and is private your will not be able to use it.

  • Inheritance: This is the ability to inherit the methods and attributes from one class to another.

  • Polymorphism: This pillar is holding hands from the inheritance one because to have polymorphism is necessary to have inheritance. For example, if you have a class called Car with the basic attributes and methods that a car could have you can create a class called Bocho or Troca that could inherit the basic methods and attributes that a car has and implement them in its own way. In this way all the cars would have the same methods but implemented in different ways or forms.

  • More Info: https://www.campusmvp.es/recursos/post/los-conceptos-fundamentales-sobre-programacion-orientada-objetos-explicados-de-manera-simple.aspx
